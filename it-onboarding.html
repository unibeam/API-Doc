Pre

Prerequisites

  • Microsoft Entra Administrator account with Global Administrator or Authentication Policy Administrator role
  • Microsoft Entra ID P1 license — required to use External Authentication Methods
  • The onboarding URL provided by the Unibeam team
  • The integration values listed above

Adding the Entra ID P1 License

Go to Microsoft 365 admin center → Billing → Your products and add Microsoft Entra ID P1.

Navigate to Billing → Your products and click Add more products
Microsoft Entra ID P1 will appear in your products list once added
6.1

Open the Onboarding Link

Your Unibeam contact will send you a unique onboarding URL. Open it in a browser while signed in as a Microsoft Entra administrator. This takes you directly to Microsoft's permission consent page for Unibeam Auth.

Note

Each URL is unique to your organization. Do not share it with others.

What you will see

Review the listed permissions and click Accept
6.2

Grant Microsoft Consent

Review the permissions and click Accept. After accepting, Microsoft redirects to the Unibeam confirmation page. Unibeam records your organization and completes the registration automatically.

Confirmation shown after successful consent — no further action needed
One-time action

This consent step applies once per organization. You will not be prompted again.

6.3

Open the Microsoft Entra Admin Center

Go to entra.microsoft.com and sign in as an administrator. Search for or navigate to Authentication methods.

Protection → Authentication methods
Search for "authentication methods" and select Microsoft Entra authentication methods
6.4

Add an External Authentication Method

In the Policies tab, click + Add external method (Preview) to open the configuration panel.

Click + Add external method (Preview)
6.5

Configure the Unibeam Method

Fill in the configuration panel using the values below. For Display Name, use a name your employees will recognise.

FieldValue
Display NameUnibeam Auth (or a name employees will recognise)
Client IDProvided by your Unibeam contact
Service Endpointhttps://ds.global.unibeam.com
App IDProvided by your Unibeam contact
Fill all four fields then click Save
6.6

Enable and Target the Method

After saving, open the method and go to the Enable and target tab. Toggle Enable to On and use + Add target to assign the groups that should use Unibeam Auth. Start with a pilot group and expand later.

Enable the toggle and assign your target groups
Example: a dedicated security group for the Unibeam rollout
Note

Only users in targeted groups will see Unibeam Auth as an MFA option. Changes may take a few minutes to propagate.

6.7

Verify User Registration Is Available

Once enabled and targeted, verify that Unibeam Auth appears as an available method for users in the assigned group by checking the authentication method panel for a test user.

Verify Unibeam SIA appears as an available external authentication method
Important

Completing this IT admin setup does not automatically register Unibeam Auth for employees. Each employee in the targeted group must individually register on their own device. See section 6.8 for the steps to share with your users.

Troubleshooting

Example: error when the onboarding URL has already been used — request a fresh URL from Unibeam
WhoIssueResolution
IT Admin Consent page shows an error Ensure you are signed in as a Global Administrator or Authentication Policy Administrator.
IT Admin External MFA option not visible in Entra Confirm your tenant has an active Entra ID P1 license. Contact your Unibeam representative.
IT Admin Authentication fails after configuration Verify the Service Endpoint and App ID are entered exactly as provided by Unibeam.
Employee Unibeam Auth not visible in security info Confirm the method is enabled and the employee's account is in a targeted group.
Employee Registration fails or times out Ensure the device has an active SIM. Wi-Fi-only devices cannot complete SIM authentication.
Employee Unibeam Auth not available at sign-in after registration Wait a few minutes for Entra to propagate group membership, then try again.

6.8 — Employee Setup Guide

Share the steps below with every employee in the targeted group. Each person completes this once on their own device.

Registration vs. ongoing sign-in

The steps below are a one-time registration. The number-match challenge only happens during this setup. After registration, every subsequent sign-in is completely silent — no codes, no notifications, no user action required.

1
Go to My Security Info
Open a browser and go to mysignins.microsoft.com/security-info. Sign in with your work account.
Click + Add sign-in method
2
Select Unibeam Auth
In the Add a method dropdown, select Unibeam Auth (or the name your IT admin configured) and click Add.
3
Approve the registration challenge
Your screen will show a 2-digit number. Enter that number in the Unibeam notification on your phone and tap Send. This is the only time you will see a challenge — future sign-ins are silent.
